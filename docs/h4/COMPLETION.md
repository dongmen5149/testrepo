## A. 베타 오픈 기준 — 가중 평가 (권장)

테스터가 **원작을 리메이크한 게임으로** 플레이한다고 가정한 가중치.  
준비(자산·데이터)는 베타 총점에 **최대 ~12%p** 만 기여 (추출 완료 ≠ 화면에 표시).

| 카테고리 | 가중치 | 완료도 | 기여 | 근거 |
|----------|--------|--------|------|------|
| **원작 동일 UI·씬** | **30%** | **~1–2%** | **~0.5%p** | Hero3: 씬 22종. H4: `MainActivity` 텍스트 PoC만, 스프라이트/맵 미렌더 |
| **시나리오·이벤트 실행** | **30%** | **~0–2%** | **~0.5%p** | SCN 350·퀘스트 128 **데이터만**; H4 앱에 SCN/퀘스트 **VM·진행기 0** |
| **게임 규칙 런타임** | **25%** | **~2–3%** | **~0.6%p** | 전투/환수/2모드/보스페이즈/인챈트 **미구현** (catalog만) |
| **본편 E2E 플레이** | **10%** | **0%** | **0%p** | 구간 클리어·베타 QA 플레이 **없음** |
| 베타 운영 (오디오·QA·스토어) | **5%** | **~5%** | **~0.3%p** | debug APK; SMAF·Play 내부 트랙·H4 회귀 테스트 없음 |
| *(상한)* 자산·데이터 준비 | *(≤12%p)* | *98–99%* | *~10–12%p* | PNG·DES·`h4_catalog` — **표시·실행 연결 전** |
| **합계 (플레이 축 + 준비 상한)** | — | — | **~12–18%** | 중앙 **~15%** |

### 왜 ~35% 보다 낮은가

| 항목 | ~35% 산정 시 | 베타 오픈 정의 시 |
|------|----------------|-------------------|
| `engine-core` + catalog PoC | KMM 50% × 12% ≈ 6%p | 테스터 체감 0 → **0%p** |
| 데이터 RE 99% | 12% 가중 거의 만점 | 화면에 안 나옴 → **상한 12%p** |
| `assembleDebug` | QA 15% × 5% | 원작 플레이 검증 아님 → **~0.3%p** |
| 씬·시스템 | 5%·3%로 과소평가했으나 | **가중 55%** 인데 **~1–2%** → 체감 거의 없음 |

### 베타 100%까지 (순서)

1. Hero3 씬 스택 이식 + **H4 맵·스프라이트 렌더** (Compose MP Step 4d + MapWalk/Battle/Dialogue)
2. **SCN·퀘스트 진행기** — 350 SCN / 128 quest 를 런타임과 연결 (Hero3 EventViewer 수준 → 본편 자동 진행)
3. **H4 전용 런타임** — 환수·2영웅×2모드·alt-form·보스 페이즈·인챈트(R106)
4. SMAF→OGG, 세이브 라운드트립, **베타 플레이 QA** (본편 구간), Play 내부 트랙
